1NF : 모든 속성 값이 원자값이어야 함 : 하나의 셀에 하나의 값만 있어야함
2NF : 부분 함수 종속 제거 : 복합 키의 일부에만 종속된 속성 제거
3NF : 이행적 함수 종속 제거 : 속성 간에 종속성이 있으면 무조건 분리하겠다.
보통 3정규화까지 함

장점 : 
중복에 따른 데이터 이상현상 최소화 > 데이터 무결성 극대화+ 저장 용량 이득
데이터 집합의 범주화 > 데이터 본질에 충실한 엔티티 도출 > 엔티티의 정체성을 가장 잘 표현하는 주 식별자 도출됨.

단점 : 
성능 저하(Overhead 존재) : 지나친 정규화는 조인 수를 늘리기 때문에 성능 저하를 유발함.
비정규화 : 정규화가 완벽하진 않다 > 성능 최적화를 위해 일부를 의도적으로 덜 정규화 된 형태로 조정할 수 있음.

DBMS 최적화
> 방향성 : 응답시간단축 및 확장성
> 도구 : 
>> 정규화 및 비정규화 : 위에 있음
>> 인덱스 추가 : 색인 속도 증가
>> 파티셔닝 : 의도적으로 분할하여 조인 데이터 축소
>> 쿼리 최적화 : 서브쿼리 등
